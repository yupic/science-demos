# Agents Notes: Runtime Конструктор Демок

Этот файл описывает, как устроен локальный конструктор демо-сцен в проекте.

## Главная идея

- Общая логика вынесена в `runtime/demo-engine.js` и `runtime/demo-engine.css`.
- Каждая демка - это отдельный HTML с минимальной обвязкой:
  - подключение runtime-файлов;
  - `sceneConfig` (структура сцены);
  - `timeline` (сценарий анимации);
  - `DemoEngine.mount(...)`.
- Не копируем большой CSS/JS между демками, меняем только конфиг и таймлайн.

## Режимы сцены

- `eqOnly`: только эквалайзеры (опционально с `controls-screen` и скоростью).
- `graphEq`: график + эквалайзеры, точки связаны со слайдерами.
- `graphOnly`: только график, точки задаются напрямую (без эквалайзеров).

## Что задается через sceneConfig

- `meta.mode`: режим (`eqOnly` / `graphEq` / `graphOnly`).
- `graph`:
  - подписи осей;
  - watermark (`РСНС`, `Аутизм` и т.д.);
  - градиент/дуга/размер графика.
- `controls`:
  - `showControlsScreen` (нужен ли экран скорости);
  - `sliderHeight` (высота ползунков в конкретной демке);
  - сетка и размеры подписей.
- `people[]`:
  - имя, цвет точки, набор слайдеров;
  - подписи слайдеров можно менять в каждой демке;
  - поддерживается `labelHtml` для многострочных подписей.
- `points[]`:
  - точки только на графике (`id`, `name`, `color`, `x`, `y`) без эквалайзеров.
- `speedPanel`:
  - `fixed` (фиксированный текст),
  - `formula2d` (скорость от x/y),
  - `weighted` (взвешенная формула для большого эквалайзера).

## Таймлайн анимации

- Простые шаги:
  - `move` - плавно переместить точку/человека к новым координатам;
  - `wait` - пауза;
  - `set` - мгновенно выставить координаты.
- Для одновременного движения нескольких точек использовать `targets` в одном `move`.
- Управление:
  - `Space` - запуск таймлайна;
  - `R` - сброс в начальное состояние.

## Практика для будущих изменений

- Сначала переиспользовать существующие шаблоны `1080p_*_constructor.html`.
- Если нужна новая демка, копировать ближайший по смыслу шаблон и менять только `sceneConfig` + `timeline`.
- Не править runtime без необходимости; сначала пытаться решить задачу конфигом.
- При правках runtime проверять, что не сломался legacy-вид `1080p_eq_only_constructor.html` (он максимально приближен к старому `1080p_eq_only.html`).
